# lidar project: downward VL53L1X rangefinder over I2C (Pi flight, cv2-free).
#
# The real read path (lidar/io/vl53l1x_reader.py :: VL53L1XReader) is a PURE
# register-level driver: it drives a bare VL53L1X over `smbus2` directly -- writes
# the 91-byte ST/Adafruit default config block, selects long distance mode @ 50 ms,
# and reads RESULT__RANGE_STATUS (0x0089) + the distance register itself. Verified
# on-device (model id EA CC 10, status 0x09, live distance). So `smbus2` is the ONLY
# sensor dependency -- no extra package, no C-extension, nothing to build. (There is
# no `pimoroni-vl53l1x` on PyPI, and the `vl53l1x` C-extension has no aarch64/py3.13
# wheel -- both irrelevant now that the register driver is the implementation.)
#
# Deviceless: the `lidar` process can still run with `--lidar-mock` (MockRangeReader,
# no sensor dep at all) -- enough to validate the 42B VIO wire + the FC UI.

# Linux SMBus / I2C transport (pure-Python, installs cleanly on the Pi5). This is
# the entire driver stack -- VL53L1XReader talks the chip's register map over it.
smbus2>=0.4.1
